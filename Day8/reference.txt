================================================================================
CRO AGENT - MARKETING SPECIALIST
================================================================================

TOOLS IMPLEMENTED:

1. get_campaign_performance(campaign_id)
   - Returns: Spend, clicks, conversions, revenue, ROAS, CPA
   
2. compare_channels(channels[])
   - Compares Facebook, Instagram, Google
   
3. calculate_customer_acquisition_cost(channel)
   - Returns: CAC, LTV/CAC ratio

KEY METRICS:

- CTR (Click-Through Rate): Clicks/Impressions × 100
- CVR (Conversion Rate): Conversions/Clicks × 100
- ROAS (Return on Ad Spend): Revenue/Spend (target: 3x+)
- CPA (Cost Per Acquisition): Spend/Conversions
- CAC (Customer Acquisition Cost): Total spend/New customers
- LTV/CAC: Lifetime Value/CAC (target: 3x+)

EXAMPLE EXECUTION:

Query: "How did Facebook Q4 campaign perform?"

Iteration 1: Call get_campaign_performance("FB_Q4_2024")
Result: Spend $250K, ROAS 2.0x, CPA $100

Iteration 2: Analyze & respond
Answer: "Facebook Q4: $250K spend, 2500 conversions, 2.0x ROAS. 
         Below 3x target - recommend optimizing ad creative."

================================================================================
AGENT ROUTER - INTELLIGENT ROUTING
================================================================================

HOW IT WORKS:

Uses GPT-3.5 to classify queries:

Input: User query
Process: LLM classifies as CFO/CRO/GENERAL
Output: Route to appropriate agent
Cost: $0.001 per routing decision

ROUTING RULES:

CFO: revenue, expenses, profit, margin, budget, forecasting
CRO: campaigns, ROAS, CAC, conversions, marketing channels
GENERAL: fallback

ROUTING ACCURACY: 95%+ (from testing)

EXAMPLES:

"What was Q4 revenue?" → CFO ✓
"Campaign performance?" → CRO ✓  
"Profit margin on Facebook ads?" → CFO ✓

WHY LLM ROUTING:
- Flexible (handles variations)
- Accurate (95%+)
- Low cost ($0.001)

Alternative (keyword matching) is brittle and fails on edge cases.
